Slide 3: What is Motion UI?

What is Motion UI?
Motion UI is a SASS library that also implements JS to bring complex transitions down to a single line of code.

What does the code for this look like?
HTML
  • <script>
  • function transitionIn(id, transition) {
  • MotionUI.animateIn("#" + id, transition);
  • }
  • </script>
  • <div id="id-1" onclick="transitionIn('id-1', 'fade-in');">Stuff like this. (click me)</div>
  • <div id="id-2" onclick="transitionIn('id-2', 'hinge-in-from-top');">And this. (click me)</div>
  • <div id="id-3" onclick="transitionIn('id-3', 'spin-in');">And this. (click me)</div>
  • <div id="id-4" onclick="transitionIn('id-4', 'scale-in-up');">And this. (click me)</div>
  • <div id="id-5" onclick="transitionIn('id-5', 'hinge-in-from-middle-x');">And this. (click me)</div>
So what is the Motion UI Library doing there?
The library is transforming those string arguments in the 'animateIn()' function, into transitions during runtime. The library supports each of those string arguments natively, as well as allowing you to create your own:
SASS
  • // SASS code
  • .mui-example {
  • @include mui-hinge(
  • $state: in,
  • $from: top,
  • $fade: true
  • );
  • }
HTML
  • <a class="secondary button" id="res-1-button" onclick="transitionIn('res-1-button', 'mui-example')">Resultant Transition</a>
Resultant Transition
What kind of transitions are possible?
  • There are animations for:
  • fade, hinge, shake, slide, spin, wiggle, & zoom
  • There are also transitions for:
  • rotation, direction, & duration
  • Transitions can be chained together or combined
